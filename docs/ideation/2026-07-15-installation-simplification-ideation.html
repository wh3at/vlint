vlint のインストール簡略化

Date: 2026-07-15 Topic: installation-simplification Focus: README Install / OS prerequisites / Browser setup Mode: repo-grounded

Grounding Context

現状は、version と repository path の置換、archive と checksum の個別取得、検証、展開、PATH への配置、22個の Ubuntu package、Playwright browser、project config の準備が別々です。しかも README は vlint init を案内せず、設定 schema v1 を掲載していますが、実装は schema v2 を生成します。

先に直すべき不整合: src/commands/check.ts:24-26 は --url 分岐より先に config を読みます。したがって README の「ad-hoc URL check は config 不要」は現行実装と一致しません。なお、更新対象は設定 schema であり、JSON出力 schema v1 は別契約なので変えません。

Topic Axes

  • System prerequisites — Ubuntu共有libraryとroot権限
  • First-run setup — browser cache、config、初回check

1. コピー可能な Quick Start を README 先頭へ

binary install→OS deps→vlint init→browser install→check --url
Description
<OWNER>/<REPO> を実在する wh3at/vlint に置換し、詳細説明より前に一つの線形ブロックを置きます。既存 vlint init を初回導線へ加え、設定例をschema v2へ同期します。providerなしのinit生成configでは引数なしcheckが targets-empty になるため、最初の成功例は vlint check --url ... にします。
Axis
First-run setup
Basis
direct: README.md:41-52, 93-107, 123-125, 140-168; src/cli.ts:84-86,119-126; src/commands/init.ts:131-135; src/commands/check.ts:24-31.
Rationale
コード変更なしで、コピー不能なplaceholder、未記載の必須コマンド、schema不整合を同時に除去できます。操作数削減より先に「書いてある通りでは動かない」状態を解消するP0です。
Downsides
詳細節との重複を持つため、release時にQuick Startも検証する仕組みが必要です。

98%
Complexity
Low

2. tag固定・user-local installer

Description
release asset として version固定の installer を配り、temp directoryでarchiveと該当checksum entryを取得・検証してから ~/.local/bin/vlint へatomic installします。標準導線は curl -fsSLO .../install.sh && sh install.sh とし、curl | sudo sh は避けます。手動検証手順は残します。
Axis
Binary acquisition
Basis
direct: README.md:40-54 の5操作を1つに畳めます。external: GitHubは固定asset名なら latest release download URL を提供しますが、再現性のためdefaultはtag固定にします。
Rationale
Node/Bun/package manager不要という製品特性を保ったまま、最も面倒な取得・検証・展開・配置を自動化します。
Downsides
~/.local/bin がPATH外なら案内が必要です。checksumは署名ではなく、scriptとarchiveが同じGitHub trust rootに属する点は現状と同じです。asset追加時は現行の単一entry SHA256SUMS 検証を見直す必要があります。
Confidence
90%
Complexity
Medium

3. vlint browser install --with-deps

Description
通常の browser install は非特権のまま維持し、明示flag時だけUbuntu packageとbrowser payloadを順に導入します。vlint自体は一般userで起動し、apt subprocessだけを昇格させ、browser cacheをroot所有にしません。
Axis
System prerequisites
Basis
direct: README.md:75-84,93-96 の二つの導入ブロックを一つにできます。external: Playwright公式にも install --with-deps があります。
Rationale
Ubuntu package listの手動コピーとPlaywright側dependency tableのdriftを減らし、PRDが禁じる check 中の暗黙導入とは分離された明示操作を保てます。
Downsides
現READMEの「OS packageを変更しない」保証をflag付き例外へ変更します。sudo・apt network・失敗途中状態の設計と検証が必要です。
Confidence
84%
Complexity
Medium

4. Ubuntu 24.04 amd64専用 .deb

Description
GitHub Releaseにchecksummed local .deb を追加し、binary配置とREADMEの共有library群を package Depends へまとめます。browser downloadはpostinstで行わず、userとしての明示的 vlint browser install を残します。
Axis
System prerequisites
Basis
direct: 対応環境は README.md:9-11 のUbuntu 24.04 x64だけです。external: Debian Policy はruntime必須packageを Depends で宣言する契約を定めます。
Rationale
sudo apt install ./vlint_..._amd64.deb 一回でsystem binaryとOS libraryを標準package transactionへ寄せられます。
Downsides
build metadata、uninstall、依存名、clean Ubuntu install、release checksumを新たに保守します。local deb自体はAPT repository署名の恩恵を受けないため、release checksum検証は残ります。
Confidence
78%
Complexity
High

5. 明示的 vlint setup

Description
vlint init と vlint browser install を一つの明示コマンドに合成します。有効な既存configはskip、未存在ならexclusive create、invalid/unreadable/symlink等はfail。途中でbrowser導入が失敗しても再実行で回復可能なidempotent挙動にします。
Axis
First-run setup
Basis
direct: 二つの機能はすでに src/cli.ts:78-86,106-126 に独立して存在し、initは src/commands/init.ts:139-189 で安全なexclusive createを行います。
Rationale
PRDの「checkとbrowser導入を別操作にする」を守りつつ、初回setupを一 invocationにします。
Downsides
project-local configとuser-global browser cacheを一コマンドに混ぜ、削減は2 invocationから1だけです。上位案より効果が小さいため最後です。
Confidence
82%
Complexity
Medium
